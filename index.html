LAB 5 -- วันที่ 22 กุมภาพันธ์ พ.ศ. 2564

เรียนรู้พื้นฐานที่ในการใช้งานโปรแกรม Burp Suite

ในแลบนี้ ให้นักศึกษาใช้ dojo VM แล้วปฏิบัติตามวิดีโอ Burpsuite Basics (FREE Community Edition) แล้วอัดวิดีโออธิบายสิ่งที่นักศึกษาทำ ตามลำดับที่ปรากฎในวิดีโอต้นแบบ

เนื่องจากใน dojo VM มีการติดตั้ง DVWA และ Burp Suite Community Edition มาเรียบร้อยแล้ว นักศึกษาจึงสามารถเริ่มปฏิบัติตามวิดึโอต้นแบบได้ตั้งแต่นาทีที่ 7:24 เป็นต้นไป

ข้อควรตระหนัก

  1. เนื่องจาก Burp Suite Community Edition ที่ได้รับการติดตั้งใน dojo VM และในเครื่องของผู้จัดทำวิดีโอต้นแบบเป็นคนละเวอร์ชั่นกัน เมนูและการทำงานบางอย่างจะไม่เหมือนกันเสียทีเดียว ให้นักศึกษาพยายามทำความเข้าใจและหาทางออกด้วยตนเองและสิ่งรอบตัว ทักษะนี้เป็นส่วนหนึ่งที่นักศึกษาต้องเรียนรู้และต้องทำการฝึกฝน

  2. ในวิดีโอมีการแสดงการโจมตีชนิด SQL Injection (sqli) ให้นักศึกษาทำตามผู้จัดทำวิดีโอ โดยยังไม่ต้องทำความเข้าใจทฤษฎีพื้นฐานเรื่องนี้ เนื่องจากจะมีการศึกษาโดยละเอียดในแลบอื่นในวิชานี้ต่อไป

  3. หากนักศึกษาสนในการใช้ Burp Suite ทำ spidering (หรือ crawling สำหรับ Burp version ปัจจุบัน) และ scanning (หรือ auditing) นักศึกษาสามารถลอง trial version ของ Burp Suite Professional Edition ได้ แต่ต้องทำการทดลองภายใน 30 วัน นักศึกษาไม่จำเป็นต้องทำในส่วนนี้สำหรับแลบนี้

สิ่งที่นักศึกษาควรเรียนรู้ในแลบนี้

  • การใช้ Burp Suite ในขั้นพื้นฐาน กล่าวคือการใช้ Tabs ต่อไปนี้คือ Target, Proxy, Repeater, Decoder, Comparer และ User options

สิ่งที่ต้องส่ง

วิดีโอ โดยให้นักศึกษาจัดระเบียบให้ครบถ้วนและมีลำดับการแสดงให้เหมือนกับที่ผู้จัดทำวิดีโอได้ทำไว้ ลองดู Handout 3 เป็นตัวอย่างได้ แต่เนื่องจากแลบนี้ค่อนข้างไม่ซับซ้อน ให้นักศึกษาเขียน script ด้วยตนเอง

LAB 3 -- วันที่ 8 กุมภาพันธ์ พ.ศ. 2564

ส่วนที่ 1: ศึกษาความแตกต่างระหว่างการแลกเปลี่ยนข้อมูลเมื่อใช้ method GET และ method POST

  1. สร้างไฟล์ form-w-get.html เพื่อแสดงแบบฟอร์มที่มีช่องรับอินพุตเป็น text เพียงหนึ่งช่อง ให้อินพุตนี้มีชื่อว่า form-input และให้ใช้ method เป็น GET
  2. สร้างไฟล์ form-w-post.html เพื่อแสดงแบบฟอร์มที่มีช่องรับอินพุตเป็น text เพียงหนึ่งช่อง ให้อินพุตนี้มีชื่อว่า form-input และให้ใช้ method เป็น POST
  3. สร้างไฟล์ get-formhandler.php และ post-formhandler.php สำหรับประมวลผลข้อมูลที่ได้รับจากการกรอกแบบฟอร์มแบบ GET และแบบ POST ตามลำดับ โดยให้แสดงผลลัพธ์เป็น form-input = x โดย x คือ ค่าที่ได้รับจากการกรอกแบบฟอร์มของผู้ใช้
  4. นำไฟล์ form-w-get.html form-w-post.html get-formhandler.php และไฟล์ post-formhandler.php ไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab2/….
  5. ทดสอบโดยใช้ burp suite และ developer tools ใน firefox
  6. พิจารณาว่าการใช้ GET และ POST นั้นมีความแตกต่างกันอย่างไร แล้วเขียนบรรยายตามความเข้าใจของตนลงในบันทึกแลบ

ส่วนที่ 2: ทดสอบการรับส่ง hidden variables

  1. ใน directory ชื่อ lab2 สร้างไฟล์ hidden-var-form.html เพื่อแสดงแบบฟอร์มที่มีช่องรับอินพุตเป็น text เพียงหนึ่งช่อง ให้อินพุตนี้มีชื่อว่า normal-var และมีการส่ง hidden variable หนึ่งตัวชื่อ hidden-var
  2. สร้างไฟล์ hidden-var-formhandler.php เพื่อประมวลผลข้อมูลที่ได้รับจากการกรอกแบบฟอร์ม โดยให้แสดงผลลัพธ์เป็น normal-var = x and hidden-var = y โดย x คือ ค่าที่ได้รับจากการกรอกแบบฟอร์มของผู้ใช้ และ y คือ ค่าของ hidden variable
  3. นำไฟล์ hidden-var-form.html และไฟล์ hidden-var-formhandler.php ไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab2/….
  4. ทดสอบโดยใช้ burp suite และ developer tools ใน firefox
  5. อธิบายว่า hidden variables คืออะไร การรับ-ส่ง hidden variable มีลักษณะอย่างไร

ส่วนที่ 3: ทดสอบการรับส่ง cookies

  1. สร้างไฟล์ชื่อ setcookie.php โดยให้มีการทำงานคือ ให้สั่ง server ให้ส่ง cookie ที่มี name เป็น remember และ value เป็น ชื่อของนักศึกษา
  2. สร้างไฟล์ชื่อ usecookie.php โดยให้มีการทำงานคือ ให้สั่ง server ให้แสดงค่า cookie ที่ได้รับมาจาก browser
  3. ใช้ burp suite และ firefox quantum ในการสำรวจเนื้อหาของข้อมูลที่มีการแลกเปลี่ยนกันสำหรับทั้งสอง requests ข้างต้น
  4. อธิบายว่า cookies คืออะไร การรับ-ส่ง cookies มีลักษณะอย่างไร

LAB 2 -- วันที่ 1 กุมภาพันธ์ พ.ศ. 2564

ขั้นตอนในการทำแลบ

แลบนี้ให้นักศึกษาใช้ dojo VM นักศึกษาต้องส่งบันทึกแลบใน MS Teams ในหมวด Assignments ตอนท้ายคาบเรียนด้วย โดยในการตอบคำถามและจดบันทึกที่จะส่งนี้ ให้เขียนฉลากแบ่งให้ชัดเจนว่า คำตอบหรือบันทึกที่ปรากฎอยู่นั้น เป็นคำตอบสำหรับส่วนใดของแลบ กรุณาจดบันทึกอย่างปราณีต และให้นักศึกษาจดในหน้าต่างที่ MS Teams เปิดให้โดยตรง ห้ามแนบไฟล์

ส่วนที่ 0: ทำให้ Virtual Box สามารถแสดง dojo VM แบบเต็มจอได้

ให้นักศึกษาสืบค้นรายละเอียดวิธีการในการทำให้ Virtual Box สามารถแสดง dojo VM แบบเต็มจอได้ ในบางกรณีโปรแกรมจะสามารถทำได้เองโดยอัตโนมัติ ในบางกรณีอาจต้องติดตั้ง Virtual Box Guest Additions

ส่วนที่ 1: ศึกษาการทำงานพื้นฐานของ HTTP

  1. Login เข้า dojo VM
  2. เปิด terminal ที่ prompt ให้พิมพ์ telnet localhost 80 รอให้ server พร้อมรับอินพุตแล้วพิมพ์ GET /index.html (case sensitive)
  3. สร้าง directory ชื่อ lab2 ใน home directory ของผู้ใช้ที่นักศึกษาเป็นอยู่ (กล่าวคือผู้ใช้ที่มี username เป็น dojo นักศึกษาสามารถตรวจสอบได้โดยการรันคำสั่ง whoami ที่ terminal prompt)
  4. copy เอาท์พุตที่ได้รับเก็บไว้ใน file ชื่อ response4getting-web-page-with-telnet.txt ใน directory ชื่อ lab2 ที่นักศึกษาได้สร้างขึ้น
  5. เปิด text editor ใดก็ได้ที่ถนัด (emacs, gedit, sublime text 3, vi, …) แล้วสร้าง HTML source file ใน directory ชื่อ lab2 โดยตั้งชื่อไฟล์ว่า hello.html ให้เนื้อหาของไฟล์คือแสดงคำว่า Hello World! ในเว็บบราวเซอร์
  6. นำไฟล์นี้ไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab2/hello.html
  7. สร้างแบบฟอร์ม hello-whoever-form.html และสร้างโปรแกรม php ชื่อ hello-whoever.php ที่สามารถแสดงคำว่า Hello xx! ในเว็บบราวเซอร์ โดย xx คือ คำที่ผู้ใช้กรอกลงไปในแบบฟอร์ม
  8. นำไฟล์ทั้งสองไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab2/hello-whoever-form.html

ส่วนที่ 2: เรียนรู้การใช้ web proxy ชื่อ Burp Suite และ firefox quantum

  1. จงตอบว่า web proxy คืออะไร มีไว้เพื่อประโยชน์อะไร โดยให้เขียนคำตอบลงในบันทึกแลบ
  2. เปิด shell แล้วพิมพ์ cd tools
  3. หาวิธีรันโปรแกรม burp suite
  4. ค้นหาข้อมูลในตัวโปรแกรม burp suite ว่ามีการตั้งค่า default ให้ฟัง port ใดอยู่
  5. กำหนดให้ firefox web browser เชื่อมต่อกับอินเทอร์เน็ตโดยผ่าน burp suite web proxy ที่ port ที่เหมาะสมก่อน โดยตั้งค่า Connection Settings
  6. ใช้ firefox เชื่อมต่อไปที่ http://targets.local หาหน้าต่างที่แสดง request และ response แล้วตอบคำถามต่อไปนี้
    1. firefox ส่ง request ไปให้ web server ทั้งหมดกี่ requests
    2. มี header ใดบ้าง ที่ถูกส่งไปในแต่ละ request
    3. response สำหรับแต่ละ request มีใจความว่าอะไร
  7. ปิดการเชื่อมต่อผ่าน burp suite สำหรับ firefox
  8. ใช้ developer tools ใน firefox ในการโหลดหน้า http://targets.local แล้วหาวิธีการเปรียบเทียบการอ่านเนื้อหาของ request และ response ตาม 3 คำถามในข้อ 6 ข้างต้น

แหล่งข้อมูลที่อาจเป็นประโยชน์สำหรับแลบนี้

ขั้นตอนในการทำแลบ

  1. ดาวน์โหลดและติดตั้งโปรแกรม Virtualbox

  2. ดาวน์โหลดและติดตั้งเครื่องเสมือน Web Security Dojo เวอร์ชั่นล่าสุด (ณ ขณะนี้เป็นเวอร์ชั่น 3.4.1) ตั้งชื่อเครื่องนี้ว่า dojo VM

  3. ดาวน์โหลด iso image ของ Ubuntu จาก Ubuntu.com มาเก็บไว้ที่เครื่อง แล้วทำการติดตั้งเป็นเครื่องเสมือนที่ถูกบริหารจัดการด้วย virtualbox ตั้งชื่อเครื่องนี้ว่า Ubuntu VM

  4. Login เข้าเครื่อง dojo VM แล้วทำ Unix lab ของ UC Berkeley ตามที่ปรากฏใน http://www-inst.eecs.berkeley.edu/~cs61a/fa14/lab/lab00/

  5. ดาวน์โหลดและติดตั้ง text editor ที่นักศึกษาต้องการใช้ใน Ubuntu VM และ dojo VM แล้วเรียนรู้วิธีการใช้ให้คล่องแคล่ว โปรแกรม editor ที่นักศึกษาอาจพิจารณาใช้ ได้แก่ Emacs, Visual Studio Code, vi, atom และ sublime text เป็นต้น

  6. สำหรับนักศึกษาที่ใช้เครื่อง Windows นักศึกษาอาจทำการตั้งค่าให้เครื่องของตนสามารถใช้งาน WSL2 ได้ด้วย จากนั้นให้ติดตั้งโปรแกรมที่ทำหน้าที่เป็น X Server (เช่น โปรแกรม X410 ที่นักศึกษาสามารถดาวน์โหลดได้จาก Microsoft Store) ในเครื่อง Windows เพื่อให้สามารถเรียกใช้โปรแกรมที่ต้องสร้างและแสดงหน้าต่างใหม่ในเครื่องได้

  7. ให้นักศึกษาเขียนบรรยายปัญหาหลัก ๆ ที่พบระหว่างการทำแลบพอสังเขปมา 3-5 ข้อ

ข้อมูลที่อาจเป็นประโยชน์

  • วิธีการตั้งค่าและติดตั้ง WSL2 และ Ubuntu ในเครื่อง Windows มีหลายวิธี นักศึกษาอาจลองวิธีที่ง่ายตามที่ปรากฏที่ https://docs.microsoft.com/en-us/windows/nodejs/setup-on-wsl2
