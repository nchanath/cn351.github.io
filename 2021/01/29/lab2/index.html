LAB 2 -- วันที่ 1 กุมภาพันธ์ พ.ศ. 2564

ขั้นตอนในการทำแลบ

แลบนี้ให้นักศึกษาใช้ dojo VM นักศึกษาต้องส่งบันทึกแลบใน MS Teams ในหมวด Assignments ตอนท้ายคาบเรียนด้วย โดยในการตอบคำถามและจดบันทึกที่จะส่งนี้ ให้เขียนฉลากแบ่งให้ชัดเจนว่า คำตอบหรือบันทึกที่ปรากฎอยู่นั้น เป็นคำตอบสำหรับส่วนใดของแลบ กรุณาจดบันทึกอย่างปราณีต และให้นักศึกษาจดในหน้าต่างที่ MS Teams เปิดให้โดยตรง ห้ามแนบไฟล์

ส่วนที่ 0: ทำให้ Virtual Box สามารถแสดง dojo VM แบบเต็มจอได้

ให้นักศึกษาสืบค้นรายละเอียดวิธีการในการทำให้ Virtual Box สามารถแสดง dojo VM แบบเต็มจอได้ ในบางกรณีโปรแกรมจะสามารถทำได้เองโดยอัตโนมัติ ในบางกรณีอาจต้องติดตั้ง Virtual Box Guest Additions

ส่วนที่ 1: ศึกษาการทำงานพื้นฐานของ HTTP

  1. Login เข้า dojo VM
  2. เปิด terminal ที่ prompt ให้พิมพ์ telnet localhost 80 รอให้ server พร้อมรับอินพุตแล้วพิมพ์ GET /index.html (case sensitive)
  3. สร้าง directory ชื่อ lab2 ใน home directory ของผู้ใช้ที่นักศึกษาเป็นอยู่ (กล่าวคือผู้ใช้ที่มี username เป็น dojo นักศึกษาสามารถตรวจสอบได้โดยการรันคำสั่ง whoami ที่ terminal prompt)
  4. copy เอาท์พุตที่ได้รับเก็บไว้ใน file ชื่อ response4getting-web-page-with-telnet.txt ใน directory ชื่อ lab2 ที่นักศึกษาได้สร้างขึ้น
  5. เปิด text editor ใดก็ได้ที่ถนัด (emacs, gedit, sublime text 3, vi, …) แล้วสร้าง HTML source file ใน directory ชื่อ lab2 โดยตั้งชื่อไฟล์ว่า hello.html ให้เนื้อหาของไฟล์คือแสดงคำว่า Hello World! ในเว็บบราวเซอร์
  6. นำไฟล์นี้ไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab2/hello.html
  7. สร้างแบบฟอร์ม hello-whoever-form.html และสร้างโปรแกรม php ชื่อ hello-whoever.php ที่สามารถแสดงคำว่า Hello xx! ในเว็บบราวเซอร์ โดย xx คือ คำที่ผู้ใช้กรอกลงไปในแบบฟอร์ม
  8. นำไฟล์ทั้งสองไปวางไว้ในที่ที่เหมาะสม เพื่อให้สามารถเข้าถึงได้โดยผ่าน URL http://targets.local/lab2/hello-whoever-form.html

ส่วนที่ 2: เรียนรู้การใช้ web proxy ชื่อ Burp Suite และ firefox quantum

  1. จงตอบว่า web proxy คืออะไร มีไว้เพื่อประโยชน์อะไร โดยให้เขียนคำตอบลงในบันทึกแลบ
  2. เปิด shell แล้วพิมพ์ cd tools
  3. หาวิธีรันโปรแกรม burp suite
  4. ค้นหาข้อมูลในตัวโปรแกรม burp suite ว่ามีการตั้งค่า default ให้ฟัง port ใดอยู่
  5. กำหนดให้ firefox web browser เชื่อมต่อกับอินเทอร์เน็ตโดยผ่าน burp suite web proxy ที่ port ที่เหมาะสมก่อน โดยตั้งค่า Connection Settings
  6. ใช้ firefox เชื่อมต่อไปที่ http://targets.local หาหน้าต่างที่แสดง request และ response แล้วตอบคำถามต่อไปนี้
    1. firefox ส่ง request ไปให้ web server ทั้งหมดกี่ requests
    2. มี header ใดบ้าง ที่ถูกส่งไปในแต่ละ request
    3. response สำหรับแต่ละ request มีใจความว่าอะไร
  7. ปิดการเชื่อมต่อผ่าน burp suite สำหรับ firefox
  8. ใช้ developer tools ใน firefox ในการโหลดหน้า http://targets.local แล้วหาวิธีการเปรียบเทียบการอ่านเนื้อหาของ request และ response ตาม 3 คำถามในข้อ 6 ข้างต้น

แหล่งข้อมูลที่อาจเป็นประโยชน์สำหรับแลบนี้

  • การ telnet เพื่อขอ web page จาก server http://support.microsoft.com/kb/279466
  • การตั้งค่า proxy เพื่อให้สามารถใช้อินเตอร์เน็ตผ่านทาง shell ได้ โดย proxy server ของทางคณะมีชื่อว่า proxy.engr และ port ที่ใช้คือ 8080 มีหลายเว็บไซต์ที่อธิบายเรื่องนี้ไว้ เช่น http://www.cyberciti.biz/faq/linux-unix-set-proxy-environment-variable/
  • การสร้างฟอร์มที่มี hidden variables http://www.w3schools.com/tags/att_input_type.asp
  • การประมวลผลข้อมูลที่ได้รับจาก form http://www.w3schools.com/php/php_forms.asp หรือ http://www.html-form-guide.com/php-form/php-form-variables.html
  • การบอก web server ให้ใส่ cookie ไว้ใน response ด้วย php http://www.w3schools.com/php/func_http_setcookie.asp
  • การใช้งาน firefox developer tools https://developer.mozilla.org/en-US/docs/Tools/Tools_Toolbox
